004 : Bank Customer Data Preparation

Comprehensive data cleaning and preparation of bank customer data for churn prediction and customer segmentation

Project Overview

This project involved preparing and cleaning a dataset of 10,000 European bank customers for machine learning applications focused on churn prediction and customer segmentation. The project demonstrates comprehensive data wrangling skills, from initial data quality assessment through to feature engineering and preparation for modeling.

The analysis helped identify key factors contributing to customer churn and provided insights for developing targeted retention strategies.

Business Context

The Bank of Mavenland was experiencing increased customer churn and declining growth. This project prepared customer data for two machine learning initiatives: predicting which customers were likely to churn and segmenting customers to better understand different behavioral patterns and needs.

Primary Question: What data preparation steps are necessary to transform raw, messy bank customer data into a clean dataset suitable for machine learning models predicting churn and identifying customer segments?

Methodology & Analysis

1 Data Import & Quality Assessment

  • Imported and joined two customer data tables from Excel using CustomerID as the key
  • Performed comprehensive data quality assessment including missing value analysis
  • Identified and removed duplicate rows and columns
  • Checked for data type inconsistencies and formatting issues
  • Documented initial data quality issues for remediation

2 Data Cleaning & Transformation

  • Fixed data type inconsistencies across all columns
  • Handled missing values appropriately for different data types (categorical vs. numerical)
  • Identified and corrected erroneous values in numerical fields
  • Standardized country names in the "Geography" column
  • Processed currency fields for consistency

3 Exploratory Data Analysis

  • Analyzed the target variable (churn) distribution
  • Explored relationships between categorical variables (Geography, Gender) and churn
  • Created box plots and histograms for numerical variables segmented by churn status
  • Identified patterns and correlations in the data
  • Visualized key insights for stakeholder communication

4 Feature Engineering & Preparation

  • Created a new "balance_v_income" feature by dividing balance by estimated salary
  • Generated dummy variables for categorical fields suitable for machine learning
  • Removed columns not suitable for modeling
  • Prepared the final dataset for machine learning applications
  • Documented the entire data preparation process

Key Findings

Data Quality Issues Identified

Issue Type Examples Resolution Approach
Missing Values Null values in categorical and numerical fields Imputed with "MISSING" for categorical, median for numerical
Inconsistent Categorization Multiple representations for same country (e.g., "FR", "France") Standardized to consistent values
Erroneous Values Negative ages, impossible credit scores Replaced with column median
Data Type Issues Numeric fields stored as text Converted to appropriate data types

Churn Insights

  • Geography Impact: Customers from Germany showed significantly higher churn rates compared to France and Spain
  • Gender Difference: Female customers had a slightly higher churn rate than male customers
  • Balance to Income Ratio: Customers with higher balance-to-income ratios were less likely to churn
  • Product Usage: Customers with more products showed lower churn rates
  • Credit Score: Customers with higher credit scores were less likely to churn

Business Impact

The prepared dataset enabled the Bank of Mavenland to:

  • Develop accurate churn prediction models to identify at-risk customers
  • Create targeted customer segments for personalized marketing
  • Understand key factors driving customer churn
  • Develop data-driven retention strategies
  • Improve data collection processes to prevent future data quality issues
Recommendation: Implement proactive retention campaigns targeting German customers and those with specific risk factors identified in the analysis. Improve data collection protocols to maintain higher data quality for future analyses.
